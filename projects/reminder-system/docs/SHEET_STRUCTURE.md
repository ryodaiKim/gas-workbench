# シート構成と設定キー

このプロジェクトは以下のシートを前提に動作します。

- 設定（設定値の保管）
- 送信ログ（送信履歴）
- 管理表: 原本（や複数の管理表* シート）
- 使い方とワークフロー（ガイド。なければ自動作成）

## 設定シートのレイアウト

2つのレイアウトに対応しています。どちらか1つで整えてください。

1) ヘッダー行レイアウト

- 1行目: キー名（見出し）
- 2行目: 値

2) 縦並び（キー・値）レイアウト

- A列: キー名
- B列: 値

どちらのレイアウトでも、キー名は以下の「必須/推奨キー名」を使用してください（完全一致）。

### 必須/推奨キー名

- 登録機関名（別名: 登録機関）
- 登録機関コード
- 送信先アドレス
- CC（任意）
- BCC（任意）
- メール件名（任意、未設定時はガイド/デフォルトを使用）
- メール本文（HTML）（任意、未設定時はガイド/デフォルトを使用）

### サポートされる別名（エイリアス）

以下の見出し名も読み替えられます。

- 登録機関 → 登録機関名
- カスタムメール件名 → メール件名
- カスタムメール本文（HTML） → メール本文（HTML）

上記以外の表記にしたい場合は、コード側にエイリアスを追加してください（`src/main.ts` の `getSettings()` 内）。

## 送信ログシート

1行目のヘッダーは次の6列に自動で整えられます。

1. 被験者ID
2. 登録医療機関
3. 送信タイミング
4. 送信先
5. 送信日時
6. 送信結果

## 管理表* シート

最低限、次の列が必要です。

- 自動送信On/Off（または 自動送信OnOff / 自動送信）
- 被験者ID
- 登録日

任意列:

- カスタム宛先（カンマ/セミコロン/改行区切り可）
	- 値がある行は、その行のTo宛先として利用されます。
	- 設定シートの送信先アドレス（機関コード別/グローバル）より優先されます。

評価・リマインドの列はリポジトリの `src/types.ts` および `src/main.ts` の定義に準じます。
